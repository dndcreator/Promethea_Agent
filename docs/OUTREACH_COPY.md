## Hacker News / Reddit Intro

We are sharing Promethea, a local-first agent runtime that focuses on runtime concerns after the first tool call.

The project currently includes a usable core around multi-user boundaries, memory backends, resumable workflows, policy/sandbox-aware execution, and trace/audit visibility. It is in public preview (not a final release), but already runnable and inspectable via Web UI, HTTP API, and CLI.

If you work on agent infrastructure and care about memory/workflow/audit/policy integration as one runtime model, feedback is welcome.

## WeChat / Technical Article Opening

Promethea 不是一个“套壳 prompt + tool call”项目，而是一个面向长生命周期 AI 系统的 local-first agent runtime。

它把记忆（memory）、工作流（workflow）、策略与沙箱（policy/sandbox）、审计与追踪（audit/trace）、多用户隔离（multi-user isolation）放在同一条 runtime 主线上处理。当前阶段是 Public Preview / Active Development：核心能力已可跑、可看、可验证，但仍在持续打磨 benchmark、打包与文档完善。

## Stage Statement (Not Released Yet, Still Worth Following)

Promethea is not positioned as a finished commercial release today.

It is a public-preview runtime project with a usable core and clear direction: local-first operation, memory/workflow continuity, policy and sandbox controls, audit visibility, and multi-user safety. If you track practical agent runtime evolution, this is a project worth watching as it matures.

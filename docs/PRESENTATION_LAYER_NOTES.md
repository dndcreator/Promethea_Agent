## Why README top was changed

The previous top section was rich in internal engineering context but less optimized for first-contact conversion. The new top keeps technical credibility while prioritizing:

1. what the project is,
2. what already works,
3. why it is different,
4. how to try it quickly,
5. current maturity status,
6. where to go next.
